# TypeScript Standards — Acceptance Criteria

**Version:** 1.0.0  
**Last Updated:** 2026-03-20

---

## AC-01: Type Safety

- [ ] Connection status enums define all valid states with TypeScript string literals
- [ ] Type definitions avoid `any` and use proper generic constraints
- [ ] Error types follow the project's AppError pattern

## AC-02: Patterns

- [ ] React component patterns follow functional component with hooks style
- [ ] State management patterns use Zustand stores with typed selectors
- [ ] API client patterns use React Query with typed query/mutation functions
