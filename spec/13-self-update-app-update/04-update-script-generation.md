# 04 — Update Script Generation

## Purpose

Define how the update-runner generates and executes a temporary script
that orchestrates the full update pipeline via `run.ps1`.

---

## Why a Generated Script?

The update-runner cannot call `run.ps1` directly from Go because:
1. `run.ps1` needs to run in a PowerShell session with proper env
2. The script needs to capture version before/after the build
3. The script needs to handle "already up to date" early-exit
4. On Windows, PowerShell execution policy must be set

---

## Windows: Generated PowerShell Script

The worker writes a temporary `.ps1` file with UTF-8 BOM encoding:

```powershell
# Auto-generated by movie update — do not edit
$ErrorActionPreference = "Stop"
$repoPath = "<repo-path>"

# Step 1: Capture current version
$oldVersion = "unknown"
$movieBin = Get-Command movie -ErrorAction SilentlyContinue
if ($movieBin -and (Test-Path $movieBin.Source)) {
    $oldVersion = (& $movieBin.Source version 2>&1) -join " "
}

# Step 2: Pull latest
Set-Location $repoPath
$pullOutput = git pull --ff-only 2>&1
$pullText = ($pullOutput | ForEach-Object { "$_" }) -join "`n"

if ($pullText -match "Already up to date") {
    Write-Host ""
    Write-Host "  Already up to date ($oldVersion)" -ForegroundColor Green
    exit 0
}

Write-Host "  Pulled new changes" -ForegroundColor Cyan
foreach ($line in $pullOutput) {
    $text = "$line".Trim()
    if ($text.Length -gt 0) { Write-Host "    $text" -ForegroundColor Gray }
}

# Step 3: Wait for parent to release file handles
Start-Sleep -Seconds 1.2

# Step 4: Build and deploy (skip pull — already done above)
& (Join-Path $repoPath "run.ps1") -NoPull

# Step 5: Compare versions
$newVersion = "unknown"
$movieBin = Get-Command movie -ErrorAction SilentlyContinue
if ($movieBin -and (Test-Path $movieBin.Source)) {
    $newVersion = (& $movieBin.Source version 2>&1) -join " "
}

Write-Host ""
if ($oldVersion -eq $newVersion) {
    Write-Host "  WARNING: Version unchanged after update" -ForegroundColor Yellow
    Write-Host "  Was version/info.go bumped?" -ForegroundColor Yellow
} else {
    Write-Host "  Updated: $oldVersion -> $newVersion" -ForegroundColor Green
}

# Step 6: Show changelog
$movieAfter = Get-Command movie -ErrorAction SilentlyContinue
if ($movieAfter -and (Test-Path $movieAfter.Source)) {
    & $movieAfter.Source changelog --latest 2>&1 | ForEach-Object { Write-Host "  $_" }
}

# Step 7: Auto-cleanup
if ($movieAfter -and (Test-Path $movieAfter.Source)) {
    & $movieAfter.Source update-cleanup 2>&1 | ForEach-Object { Write-Host "  $_" }
}
```

## Unix: Direct Shell Execution

On Linux/macOS, the worker runs `run.ps1` via `pwsh` (if available) or
falls back to a simpler bash approach:

```go
func executeUpdateUnix(repoPath string) {
    // Option 1: pwsh run.ps1
    if _, err := exec.LookPath("pwsh"); err == nil {
        runPS1 := filepath.Join(repoPath, "run.ps1")
        cmd := exec.Command("pwsh", "-File", runPS1, "-NoPull")
        // ...
    }

    // Option 2: Direct go build
    // git pull → go mod tidy → go build → copy to deploy path
}
```

---

## Script Encoding

The generated PowerShell script MUST be written with **UTF-8 BOM**
(`0xEF 0xBB 0xBF`) prefix. Without BOM, PowerShell may misinterpret
Unicode characters (emoji in output messages).

---

## Script Cleanup

The temp script is deleted after execution:
- Written to: `%TEMP%/movie-update-script-*.ps1`
- Deleted by: `defer os.Remove(scriptPath)` in the Go code

---

## No Interactive Prompts

The generated script MUST NOT contain:
- `Read-Host` calls
- `Pause` commands
- Any interactive input

The script runs in a non-interactive PowerShell session. If `run.ps1`
encounters conflicts (e.g., local changes), the update must fail cleanly
rather than hang waiting for input.
